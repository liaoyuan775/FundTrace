# FundTrace 公安蓝白主题设计

## 目标

保留当前页面结构、交互和信息密度，把红黑主导的视觉主题替换为更符合公安客户偏好的深蓝导航与浅色业务内容。蓝色承担品牌、导航、选择和主要操作语义；红色只承担高风险、错误和必须关注的异常语义。

## 色彩结构

- 顶部栏和左侧主导航使用深公安蓝，形成稳定的应用框架。
- 案件筛选、研判图区、检查器、材料与校核工作区使用白色和浅灰蓝背景。
- 分隔线、输入框和次级背景使用中性灰蓝，避免页面变成单一蓝色。
- 主按钮、当前导航、选中节点、选中路径和焦点状态使用标准行动蓝。
- 入账与出账路径使用两种有区分度的蓝色；普通关系使用低饱和灰蓝。
- 高风险分数、风险标签、错误和冲突继续使用红色；中风险使用橙色；成功状态保留绿色。

## 页面应用

- 顶栏、左侧导航和品牌区保留深色，但从近黑色改为深蓝。
- 过滤器、右侧检查器和拓扑背景改为浅色；拓扑网格使用很淡的蓝灰线。
- 原始流水表格维持浅色结构，表头、悬停和选中态统一改为蓝灰与浅蓝。
- 按钮、输入框、下拉框、标签、提示条和滚动条统一到同一套蓝白层级。
- 不改变尺寸、间距、排版、文案、图标、页面分区或操作流程。

## 拓扑语义

- 普通节点使用中蓝，关键关联节点使用深蓝，高风险节点保留红色。
- 主干路径使用清晰的灰蓝实线，辅助路径使用更淡的蓝灰曲线。
- 入度、出度、选中路径和播放状态采用不同明度的蓝色，确保仍能快速区分。
- 标签背景改为白色或极浅蓝，文字使用深蓝灰，提高浅色图面的可读性。

## 实现边界

- 优先调整 `frontend/src/styles.css` 的主题变量和相关硬编码颜色。
- 同步调整 `frontend/src/GraphCanvas.tsx` 中无法使用 CSS 变量的画布颜色。
- 不修改组件结构、业务逻辑、后端接口、数据模型或端口配置。
- 当前前端端口继续固定为 `5173`，后端端口继续固定为 `8000`。

## 验证标准

- 自动化测试确认拓扑颜色仍区分普通、风险、入度、出度和选中状态。
- 现有前端交互测试全部通过，生产构建成功。
- 浏览器检查案件、材料、校核、起点和资金研判页面，确认无残留的大面积红黑主题。
- 桌面首屏中文字、表格、控件和拓扑标签对比度清晰，没有颜色导致的遮挡或不可读内容。
