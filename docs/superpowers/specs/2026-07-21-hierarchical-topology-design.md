# FundTrace 分层树状拓扑体验设计

## 目标

在不隐藏任何账户或转账路径的前提下，把当前自由力导向图改为从左到右展开的分层树状拓扑，降低节点、标签和连线交叉造成的阅读负担。同时让用户在拓扑区域任意位置按住拖动时，都能稳定平移整张图。

## 排布规则

- 根据交易边的付款方和收款方计算节点层级，优先使用入度为零的账户作为根节点。
- 从根节点沿资金方向计算最短层级；循环、回流或孤立子图使用确定性的兜底规则分配层级，确保相同数据每次得到相同排布。
- 节点从左到右按层级排列；同层节点根据相邻层重心排序，减少主干路径交叉。
- 每个非根节点选择一条主要入边作为树干。主要入边优先选择来自上一层、首次交易时间更早的路径，再以金额和路径 ID 保证稳定排序。
- 所有节点和所有边继续展示。树干使用清晰的实线和箭头；回流、跨层和额外入边使用更淡的弯曲辅助线，避免和树干争夺视觉注意力。
- 节点标签保留，但与节点保持固定间距，避免标签压在线上。

## 交互规则

- 拓扑区域中的画布、节点和连线统一支持按住拖动平移视图。
- 取消单个节点自由拖动，防止节点拖动与画布平移竞争，也避免人工拖乱稳定树状排布。
- 空闲状态显示抓手光标，拖动时显示抓取光标。
- 滚轮缩放、节点点击、连线点击、表格联动、播放和复位选择保持现有行为。
- 拖动手势不得误触节点或连线点击。

## 实现边界

- 层级计算和树干选择放在独立的纯函数模块中，输入为现有 `GraphData`，输出为带固定坐标及主干边集合的布局结果。
- `GraphCanvas` 使用预设坐标渲染，不再运行随机力导向模拟。
- 不修改后端图接口，不增加数据库字段，不隐藏或合并交易路径。
- 不改变当前固定端口：前端 `5173`，后端 `8000`。

## 验证标准

- 单元测试覆盖多根节点、分支、归集、回流和孤立节点，验证层级、坐标与主干选择稳定。
- 图配置测试确认使用预设坐标、区分主干和辅助边，并启用全区域画布拖动、禁用节点拖动。
- 现有节点、连线和表格选择测试继续通过。
- 浏览器验证 52 个账户和 117 条路径全部渲染，拓扑从左到右分层展开；在空白、节点和连线区域开始拖动均能平移视图；点击查看逻辑不回归。
